ubuntu 20.04 安装配置 Nginx

发表于 2024-09-17|更新于2024-09-17|Linux
|字数总计:1735|阅读时长:6分钟|阅读量:

1. 更新系统

在安装任何新软件之前，最好先更新你的系统包索引

sudo apt update
sudo apt upgrade

2. 安装 Nginx

使用 apt 包管理工具来安装 Nginx

sudo apt install nginx

3. 启动 Nginx

安装完成后，Nginx 应该会自动启动。可以使用以下命令来检查 Nginx 的状态

sudo systemctl status nginx

状态显示 active (running)，且有两个工作进程 (worker process)

表示 Nginx 已成功启动并正在运行

如果 Nginx 没有运行，你可以手动启动它

sudo systemctl start nginx

要确保 Nginx 在系统启动时自动启动，可以使用以下命令

sudo systemctl enable nginx

输入后出现以下内容表示设置成功
Synchronizing state of nginx.service with SysV service script with /lib/systemd/systemd-sysv-install.
Executing: /lib/systemd/systemd-sysv-install enable nginx

具体解释：

  • Synchronizing state of nginx.service with SysV service script：表示系统正在将 nginx 服务的启动状态与 SysV 脚本同步。这是在旧版服务管理方式和现代 systemd 之间的一个兼容性操作。

  • Executing: /lib/systemd/systemd-sysv-install enable nginx：表示实际执行的命令，用于将 nginx 服务配置为开机自动启动。

想检查这一设置是否生效，可以使用以下命令查看服务的启动状态

sudo systemctl is-enabled nginx

如果输出是 enabled，则表示 Nginx 服务已成功配置为在启动时自动启动。如果输出是 disabled，则表示自动启动设置没有成功。

5. 测试 Nginx

打开浏览器，访问你的服务器 IP 地址或域名。你应该能看到 Nginx 的默认欢迎页面。如果你看到这个页面，说明 Nginx 已成功安装并在运行。

6. 配置 Nginx 服务器块

本文不包括nginx 具体配置，请自行查询

为了托管多个站点，或者设置特定域名的配置，可以创建和配置 Nginx 服务器块

可以进入 /etc/nginx/sites-available/ 目录下修改你的服务器块 default

在 Nginx 的配置目录 /etc/nginx/sites-available/ 中，default 文件是 Nginx 默认的服务器块配置文件。它为没有配置自定义域名或服务器块的请求提供默认的处理方式。默认情况下，它用于处理发往服务器的所有 HTTP 请求

作用

  1. 默认站点配置：default 文件定义了当访问服务器时，如果请求的域名没有匹配任何其他服务器块时，该如何处理请求。Nginx 会将这些请求路由到 default 文件配置的站点。

  2. 新安装时的默认站点：在新安装的 Nginx 中，default 通常配置了一个简单的欢迎页面，这样你可以快速测试 Nginx 是否工作。

什么时候修改 default 文件？

  1. 自定义默认站点：如果你想让服务器处理没有指定域名的请求，或者你只需要一个站点，那么可以直接修改 default 文件。

  2. 不需要默认站点：如果你已经配置了多个自定义服务器块，可以选择禁用或删除 default 文件的符号链接（位于 /etc/nginx/sites-enabled/）来避免默认站点干扰你的自定义站点。

禁用 default：

sudo rm /etc/nginx/sites-enabled/default

重启 nginx 生效

sudo systemctl reload nginx

default 文件是 Nginx 安装时的一个示例配置文件，用来处理未指定的域名请求。如果你已经设置了其他站点，通常可以删除或修改这个文件。

当然也可以创建新的服务器块 sudo nano /etc/nginx/sites-available/example.com

启用服务器块配置：

在 Nginx 中，只有 sites-enabled 目录下的配置文件或符号链接会被实际使用。你需要为每个站点创建符号链接，将其链接到 sites-enabled。

启用 example.com

sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/

测试 Nginx 配置： 在重新加载 Nginx 之前，测试配置文件是否正确

sudo nginx -t

如果测试结果为 syntax is ok 和 test is successful，说明配置没有问题。

重新加载 Nginx： 如果没有错误，重新加载 Nginx 使更改生效

sudo systemctl reload nginx

7. 部署阿里云SSL

步骤 1：从阿里云下载 SSL 证书

  1. 登录阿里云控制台。
  2. 进入 SSL 证书服务。
  3. 找到你申请的 SSL 证书并下载，选择 Nginx 格式。
  4. 你将会下载到一个压缩包，里面包含两个主要文件：
    • example.com.pem（包含公钥证书）
    • example.com.key（包含私钥）

将这些文件保存到你的服务器上，比如 /etc/nginx/ssl/ 目录中。

步骤 2：将证书上传到服务器

将下载好的证书上传到服务器上的适当位置，比如 /etc/nginx/ssl/。

  1. 通过 SFTP、FTP 或 SCP 将证书文件上传到服务器。
  2. 确保文件权限正确，以防止未经授权的用户访问私钥文件：

sudo mkdir -p /etc/nginx/ssl
sudo cp /path/to/example.com.pem /etc/nginx/ssl/
sudo cp /path/to/example.com.key /etc/nginx/ssl/
sudo chmod 600 /etc/nginx/ssl/example.com.key
sudo chmod 644 /etc/nginx/ssl/example.com.pem

步骤 3：修改 Nginx 配置文件

现在你需要在 Nginx 的配置中引用 SSL 证书。

  1. 打开你的网站的 Nginx 配置文件（假设站点为 default）：

sudo nano /etc/nginx/sites-available/default

  1. 在现有的 server 块中添加 SSL 配置，或者创建一个新的服务器块来处理 HTTPS 请求：

解释：

  • listen 443 ssl：告诉 Nginx 监听 HTTPS 请求的 443 端口，并启用 SSL。
  • ssl_certificate：这是公钥证书的路径。
  • ssl_certificate_key：这是私钥的路径。
  • ssl_protocols：启用的 SSL/TLS 协议版本，建议仅启用 TLS 1.2 和 1.3。
  • ssl_ciphers：指定允许的密码套件，确保安全性。
  • server_name：你的域名或站点名称。
  • 重定向 HTTP 到 HTTPS：通过 301 永久重定向所有 HTTP 流量到 HTTPS。

步骤 4：测试 Nginx 配置

在重新加载 Nginx 之前，先测试配置是否正确：

sudo nginx -t

如果一切正常，你应该看到类似 syntax is ok 和 test is successful 的输出。

步骤 5：重新加载 Nginx

重新加载 Nginx 以应用新的配置：

sudo systemctl reload nginx

步骤 6：开放 443 端口

确保在阿里云安全组中开放 443 端口：

  1. 登录阿里云控制台。
  2. 进入 云服务器 ECS -> 网络与安全 -> 安全组。
  3. 找到对应的安全组，点击 配置规则。
  4. 添加入站规则，允许 443 端口通过 TCP 协议接收来自 0.0.0.0/0 的请求（所有 IP）

验证

现在你可以在浏览器中访问 https://default.com，检查是否成功启用了 HTTPS。
